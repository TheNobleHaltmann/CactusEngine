Team Cacti

Introduction

The Cactus Engine is a 2D engine for creating games using an HTML canvas and Javascript.

Features

Object oriented

Each game object can be assigned its own sprite, update function, draw function, etc.

Sprite sheet animation

Sprites can be static images or sprite sheets

Other objects

In addition to sprites, engine includes classes for drawing straight lines and writing text onto the canvas.

Input handling

Each object can be assigned its own function in response to mouse events and/or keyboard events. When input occurs, all objects with the relevant response function are notified.

Collision detection with axis aligned bounding boxes

Objects are only checked for collision if they have been assigned a collision function.

Define update frequency

Set how often the game updates on starting the engine.

Networking via Peer.js

Inputs sent from one device to the other.

ADDING OBJECTS

To add an object to the engine, first create a drawnObject, then add the drawnObject to the engine using engine.addObject():

The drawnObject constructor includes:

  • type: A string value that is not used by the engine at all. Intended to be used by the developer to determine what an object is.
  • width: The width in pixels of the image. Must be an integer.
  • height: The height in pixels of the image. Must be an integer.
  • x: The x coordinate of the center of the image on the canvas. Must be an integer.
  • y: The y coordinate of the center of the image on the canvas. Must be an integer.
  • imageSrc: A string value containing the path to find the image. An example is: "./cops/cop.png".

REMOVING OBJECTS

To remove an object from the engine, use engine.removeObject( obj ). To keep an object in the engine but remove anything that has been bound to it, use engine.unbindEvents( obj ).
